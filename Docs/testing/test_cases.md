# Тест-кейсы TaMP

## TC-AUTH — Авторизация

| ID | Предусловие | Входные данные | Ожидаемый ответ | Тип |
|---|---|---|---|---|
| TC-AUTH-01 | Пользователь зарегистрирован (login=alice, hash=hash1) | `auth&alice&hash1` | `auth+&Alice` | Позитивный |
| TC-AUTH-02 | Пользователь зарегистрирован | `auth&alice&wronghash` | `auth-` | Негативный |
| TC-AUTH-03 | Пользователь не зарегистрирован | `auth&nobody&hash` | `auth-` | Негативный |
| TC-AUTH-04 | — | `auth&alice` (нет пароля) | `auth-` или нет ответа | Граничный |

---

## TC-REG — Регистрация

| ID | Предусловие | Входные данные | Ожидаемый ответ | Тип |
|---|---|---|---|---|
| TC-REG-01 | Логин не занят | `reg&bob&Bob&hashb&bob@test.com` | `reg+&bob` | Позитивный |
| TC-REG-02 | Логин уже занят | `reg&bob&Bob2&hashb2&bob2@test.com` | `reg-` | Негативный |
| TC-REG-03 | — | `reg&bob&hashb` (неполные данные) | нет ответа / `reg-` | Граничный |

---

## TC-RESET — Сброс пароля

| ID | Предусловие | Шаг | Входные данные | Ожидаемый ответ | Тип |
|---|---|---|---|---|---|
| TC-RST-01 | Пользователь зарегистрирован | 1 | `reset&alice` | `reset+&***** ce@mail.ru` | Позитивный |
| TC-RST-02 | Пользователь не существует | 1 | `reset&nobody` | `reset-` | Негативный |
| TC-RST-03 | Код отправлен | 2 | `confirm&alice&{code}` | `confirm+` | Позитивный |
| TC-RST-04 | Неверный код | 2 | `confirm&alice&000000` | `confirm-` | Негативный |
| TC-RST-05 | Код подтверждён | 3 | `newpass&alice&newhash` | `newpass+` | Позитивный |
| TC-RST-06 | После смены пароля | — | `auth&alice&newhash` | `auth+&Alice` | Верификация |

---

## TC-INT — Сквозные тесты (ручные через PuTTY)

| ID | Сценарий | Шаги | Ожидаемый результат |
|---|---|---|---|
| TC-INT-01 | Полный цикл: регистрация → вход | reg → auth | Успешный вход |
| TC-INT-02 | Сброс пароля и повторный вход | reset → confirm → newpass → auth | Вход с новым паролем |
| TC-INT-03 | Работа сервера в Docker | docker run → telnet → auth | Ответ от контейнера |
