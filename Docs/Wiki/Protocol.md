# Протокол TaMP

Протокол текстовый, кодировка UTF-8. Каждое сообщение заканчивается символом `\n`.  
Разделитель аргументов внутри сообщения — `&`.

## Команды

### Авторизация

| Направление | Формат |
|---|---|
| Запрос | `auth&{login}&{md5_password}` |
| Успех  | `auth+&{display_name}` |
| Ошибка | `auth-` |

Пароль передаётся как MD5-хэш строки, введённой пользователем.

---

### Регистрация

| Направление | Формат |
|---|---|
| Запрос | `reg&{login}&{name}&{md5_password}&{email}` |
| Успех  | `reg+&{login}` |
| Ошибка | `reg-` |

Ошибка возвращается, если логин уже занят.

---

### Сброс пароля (3 шага)

**Шаг 1 — запрос кода:**

| Направление | Формат |
|---|---|
| Запрос | `reset&{login}` |
| Успех  | `reset+&{masked_email}` — код отправлен на email |
| Ошибка | `reset-` |

**Шаг 2 — подтверждение кода:**

| Направление | Формат |
|---|---|
| Запрос | `confirm&{login}&{code}` |
| Успех  | `confirm+` |
| Ошибка | `confirm-` |

**Шаг 3 — установка нового пароля:**

| Направление | Формат |
|---|---|
| Запрос | `newpass&{login}&{new_md5_password}` |
| Успех  | `newpass+` |
| Ошибка | `newpass-` |
